工具层：jieba 中文分词

jieba 是什么 工具

jieba（结巴）是 Python 中最主流的中文分词工具，由 Sun Junyi 开发。它基于前缀词典 + HMM 模型实现，特点是开箱即用、速度较快、支持自定义词典。

三种分词模式

模式函数特点适用场景
精确模式jieba.cut(s, cut_all=False)试图将句子最精确地切开，适合文本分析默认首选
全模式jieba.cut(s, cut_all=True)把句子中所有可以成词的词语都扫描出来，速度快但不能消除歧义搜索引擎构建索引
搜索引擎模式jieba.cut_for_search(s)在精确模式基础上，对长词再次切分搜索引擎分词

核心原理：词典 + HMM

jieba 内部是一个混合策略：

  1. 前缀词典匹配：先基于内置的 50 万+ 词条词典，用有向无环图（DAG）找出所有可能的切分路径，然后用动态规划选概率最大路径
  2. HMM 新词发现：对于词典中没有的词（未登录词），用 HMM 模型做新词识别。隐状态为 {B, M, E, S}，维特比解码得到最优标签序列

关键用法

Python 
import jieba

# 基础分词
text = "文本智能处理是NLP的核心任务"
words = jieba.lcut(text)  # ['文本', '智能', '处理', '是', 'NLP', '的', '核心', '任务']

# 添加自定义词典（解决领域专有词汇）
jieba.add_word('文本智能')
jieba.load_userdict('my_dict.txt')  # 每行一个词

# 关键词提取
import jieba.analyse
keywords = jieba.analyse.extract_tags(text, topK=3)

# 词性标注
import jieba.posseg as pseg
for word, flag in pseg.cut(text):
    print(f"{word}: {flag}")  # n(名词), v(动词), a(形容词)...
入门建议：先掌握 jieba.lcut() 精确模式，然后学会添加自定义词典解决领域词汇。词性标注和关键词提取是加分项。不需要深入 jieba 内部的 HMM 源码，但要知道它靠 HMM 处理未登录词。

表示层：Word2Vec 词向量

为什么需要 Word2Vec 核心问题

神经网络只认识数字，不认识文字。Word2Vec 解决的是：如何把词语映射成神经网络能处理的稠密向量，同时保留语义关系。

两种训练架构

CBOW（连续词袋）

用上下文词预测中心词

输入：「我」「喜欢」「深度」「学习」→ 预测：「自然语言」

适合：小型语料库

Skip-gram

用中心词预测上下文词

输入：「自然语言」→ 预测：「我」「喜欢」「深度」「学习」

适合：大型语料库，更常用

核心机制：预测任务制造语义向量

Word2Vec 的巧妙之处在于：它不是直接「定义」词向量，而是通过一个预测任务「逼」神经网络学到好的表示。

训练过程：

  1. 每个词用两个向量表示：中心词向量 v(w) 和上下文词向量 u(w)
  2. 在语料中滑动窗口，不断制造（上下文 → 目标词）的训练样本
  3. 神经网络的目标是最小化预测误差，在此过程中自动调整词向量
  4. 语义相近的词（如「国王」和「女王」）被迫在向量空间中靠近，因为它们出现在相似的上下文中

最著名的特性：向量类比

v("国王") - v("男人") + v("女人") ≈ v("女王")

这不是人为设计的规则，是训练过程的副产品。模型发现「国王」和「女王」的上下文分布差异，恰好与「男人」和「女人」的差异方向一致。这让词向量从"编码"变成了"有意义的表示"。

Word2Vec 的局限

  • 上下文无关：「bank」在「river bank」和「bank account"」中是同一个向量——这是它被 ELMo、BERT 取代的根本原因
  • 未登录词：训练集中没出现过的词没有向量（KeyError）
  • 多义词困境：一个词只能有一个向量，无法表示多个语义

模型层：FNN → RNN → LSTM 演进

这三者的演进逻辑非常清晰：文本是序列数据，FNN 处理不了序列，RNN 能处理但有致命缺陷，LSTM 修复了缺陷。每一代都是为解决上一代的核心问题而生。

FNN（前馈神经网络）：为什么不适合文本 不适合序列

FNN 的结构是「输入 → 隐藏层 → 输出」，信息流单向、无循环。

致命缺陷：

  • 输入维度固定：一个句子 10 个字，另一个 100 个字，FNN 无法自适应处理不同长度的输入
  • 无记忆能力：处理第 5 个字时，网络已经"忘记"了前 4 个字的信息
  • 位置无关：把词向量拼接后输入，词序信息丢失——「我不喜欢他」和「他喜欢不我」变成相同的输入

RNN（循环神经网络）：引入记忆 核心改进

RNN 的核心设计：隐藏状态 hₜ 不仅依赖当前输入 xₜ，还依赖上一时刻的隐藏状态 hₜ₋₁。这相当于网络有了一个「记忆」，信息可以在时间步之间传递。

  • 处理 x₁ 后产生 h₁，h₁ 编码了 x₁ 的信息
  • 处理 x₂ 时，h₂ 同时接收 x₂ 和 h₁，因此 h₂ 编码了 x₁ 和 x₂ 的信息
  • 以此类推，hₜ 理论上编码了 x₁ 到 xₜ 的全部历史

RNN 的致命缺陷：梯度消失/爆炸

数学根源：RNN 反向传播时，梯度需要通过时间步长链式相乘。假设激活函数导数最大为 1，经过 T 个时间步，梯度会乘以 W 的 T 次方。如果 W 的特征值 < 1，梯度指数级衰减（消失），网络无法学习长距离依赖；如果 W 的特征值 > 1，梯度指数级爆炸，训练不稳定。

实际后果：RNN 只能有效捕捉约 5-10 个时间步的依赖关系。对于一个 50 个字的句子，开头和结尾之间的语义关联 RNN 根本学不到。

LSTM（长短期记忆网络）：解决长距离依赖 当前主流

LSTM 的核心思想：不是让隐藏状态全部参与传递，而是引入一个专门的「细胞状态」（Cell State）作为信息高速公路，并通过门控机制决定哪些信息保留、哪些遗忘、哪些更新。

三个门 + 一个细胞状态

组件公式作用
遗忘门 fₜσ(W_f · [hₜ₋₁, xₜ] + b_f)决定从细胞状态中丢弃哪些旧信息（0=全忘，1=全留）
输入门 iₜσ(W_i · [hₜ₋₁, xₜ] + b_i)决定哪些新信息写入细胞状态
候选状态 C̃ₜtanh(W_c · [hₜ₋₁, xₜ] + b_c)生成新的候选信息
细胞状态 Cₜfₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ信息高速公路：旧信息选择性保留 + 新信息选择性加入
输出门 oₜσ(W_o · [hₜ₋₁, xₜ] + b_o)决定从细胞状态中输出什么到隐藏状态 hₜ
LSTM 为什么能缓解梯度消失？

细胞状态的更新公式 Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ 是一个加法路径，而非 RNN 的乘法路径。反向传播时，梯度通过细胞状态传递时是逐元素相加（+），而非相乘（×）。这意味着即使经过 100 个时间步，梯度也不会指数级衰减——信息可以在长距离上稳定流动。

三者对比总结

维度FNNRNNLSTM
序列处理能力无有（隐藏状态传递）有（细胞状态 + 门控）
变长输入不支持支持支持

应用层：中文情感分析完整 Pipeline

什么是情感分析

情感分析（Sentiment Analysis）是判断文本情感极性的任务。二分类版本：判断一段文本是正面还是负面。例如：

  • 「这部电影太精彩了，强烈推荐！」→ 正面
  • 「完全浪费时间，剧情毫无逻辑」→ 负面

→

→

正面: 0.92 / 负面: 0.08

两种建模方案

方案架构优点缺点
词袋 + FNN词向量平均 → FNN → Softmax简单快速，不需要序列模型丢失词序信息，「不很好」=「很好」
LSTM 序列建模词向量序列 → LSTM → 取最后时刻 h → Softmax保留词序，能处理否定、程度副词训练较慢，需要更多数据

关键设计决策

  • 词向量选择：小数据用预训练 Word2Vec（如知乎/微博语料训练的 SGNS），大数据可端到端训练
  • OOV 处理：未登录词用「UNK」向量或字符级 CNN 辅助
  • 否定处理：「不太好」vs「很好」——LSTM 能通过门控学习「不」对后续词的情感翻转
  • 程度副词：「非常好」vs「还行」——LSTM 能捕捉程度词的修饰关系

数据集推荐

  • ChnSentiCorp：携程酒店评论，约 1 万条，二分类，中文情感分析经典入门数据集
  • weibo_senti_100k：微博评论，约 10 万条，规模较大
  • online_shopping_10_cats：10 类商品评论，多领域
入门路径建议：
1. 先用 jieba + Word2Vec + FNN 跑通基线（理解 Pipeline 全貌）
2. 再把 FNN 换成 LSTM，观察词序敏感任务的提升
3. 最后尝试预训练 BERT 微调，体验现代范式的威力
这个递进过程能让你理解每一层技术升级的价值。

完整 Pipeline 代码

方案 A：jieba + Word2Vec + FNN（基线）

Python / PyTorch 
import jieba
import torch
import torch.nn as nn
from gensim.models import Word2Vec
from torch.utils.data import Dataset, DataLoader

# ========== 1. 数据准备 ==========
texts = ['产品质量很好', '体验太差了', '物流很快', '完全不值这个价']
labels = [1, 0, 1, 0]  # 1=正面, 0=负面

# jieba 分词
tokenized = [jieba.lcut(t) for t in texts]

# 训练 Word2Vec（实际用预训练模型更好）
w2v = Word2Vec(tokenized, vector_size=100, min_count=1, sg=1)

# 文本 → 向量（平均池化）
def text_to_vec(tokens, model, dim=100):
    vecs = [model.wv[t] for t in tokens if t in model.wv]
    return sum(vecs) / len(vecs) if vecs else [0]*dim

X = torch.tensor([text_to_vec(t, w2v) for t in tokenized], dtype=torch.float32)
y = torch.tensor(labels, dtype=torch.long)

# ========== 2. FNN 模型 ==========
class FNNClassifier(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_classes):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, num_classes)
        )
    def forward(self, x):
        return self.net(x)

model = FNNClassifier(100, 64, 2)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

# ========== 3. 训练 ==========
for epoch in range(100):
    optimizer.zero_grad()
    outputs = model(X)
    loss = criterion(outputs, y)
    loss.backward()
    optimizer.step()
    if epoch % 20 == 0:
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")

方案 B：jieba + Word2Vec + LSTM（推荐）

Python / PyTorch 
# ========== 1. 数据准备（需要固定长度 / Padding） ==========
from torch.nn.utils.rnn import pad_sequence

# 构建词汇表
vocab = {'<PAD>': 0, '<UNK>': 1}
for tokens in tokenized:
    for t in tokens:
        if t not in vocab:
            vocab[t] = len(vocab)

# 文本 → 索引序列
def tokens_to_ids(tokens, vocab, max_len=20):
    ids = [vocab.get(t, vocab['<UNK>']) for t in tokens]
    ids = ids[:max_len]  # 截断
    ids += [vocab['<PAD>']] * (max_len - len(ids))  # 填充
    return ids

X_ids = torch.tensor([tokens_to_ids(t, vocab) for t in tokenized])
y = torch.tensor(labels, dtype=torch.long)

# 嵌入矩阵：用 Word2Vec 初始化
embedding_matrix = torch.randn(len(vocab), 100)
for word, idx in vocab.items():
    if word in w2v.wv:
        embedding_matrix[idx] = torch.tensor(w2v.wv[word])

# ========== 2. LSTM 模型 ==========
class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.embedding.weight.data.copy_(embedding_matrix)
        self.lstm = nn.LSTM(embed_dim, hidden_dim,
                            batch_first=True, bidirectional=True)
        self.fc = nn.Linear(hidden_dim * 2, num_classes)

    def forward(self, x):
        embedded = self.embedding(x)           # (batch, seq, embed)
        lstm_out, (hidden, cell) = self.lstm(embedded)
        # 取最后时刻的隐藏状态（双向拼接）
        hidden = torch.cat((hidden[-2], hidden[-1]), dim=1)
        return self.fc(hidden)

model = LSTMClassifier(len(vocab), 100, 64, 2)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

# ========== 3. 训练 ==========
for epoch in range(100):
    optimizer.zero_grad()
    outputs = model(X_ids)
    loss = criterion(outputs, y)
    loss.backward()
    optimizer.step()
    if epoch % 20 == 0:
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")

# ========== 4. 预测 ==========
model.eval()
with torch.no_grad():
    test_text = "服务态度非常好"
    test_tokens = jieba.lcut(test_text)
    test_ids = torch.tensor([tokens_to_ids(test_tokens, vocab)])
    pred = model(test_ids).argmax(dim=1)
    print(f"'{test_text}' → {'正面' if pred.item() == 1 else 'negative'}")
关键代码要点：
1. Padding：LSTM 需要固定长度输入，短句补 <PAD>，长句截断
2. 预训练初始化：用 Word2Vec 初始化 nn.Embedding，比随机初始化效果好很多
3. 双向 LSTM：bidirectional=True 同时看前文和后文，情感分析必开
4. 隐藏状态拼接：双向 LSTM 的最后时刻隐藏状态是 [前向最后, 后向最后]，维度要 ×2
